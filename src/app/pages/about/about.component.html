Populating Question Database

For Paper Generator App

This website is meant to be used for populating the database of questions. The objective is to create a huge repository of questions - as much as 6000 questions over different topics of high school physics. It will be a collection of questions from different books, assignments and DPPs of different institutes & schools. It would also include questions which actually came in previous exams. However, I would like to minimize repetition of questions in the database.
Finally, one can query this database, to create assignments, exams, etc of varying difficulty level dynamically.

There will be need to understand the design of the database. I think for people who are just learning Database design, this can be a good small project. At least, it was for me.

So, here we go:
Populating the database starts with outermost tables in my Entity diagram - tables which are independent and have no parent.
Question Types, Subjects, Difficulty Levels, Images and Tags are these type of tables.
Next, there might be Images for both Questions and Answers - which we need to enter first, alongwith their sizes to be used in the LaTeX markup.
Next, we input the Question Text , followed by Answer Text, for each option of the Answer.
QuestionTags is populated last and may be done at a later stage. I put this entire data entry flow within a transaction to avoid concurreny.
